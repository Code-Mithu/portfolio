# Information Architecture (IA.md)

## 1. Site Map
*   **Home**
*   **Projects** (List View)
    *   Project Details (Dynamic)
*   **Experience**
*   **About**
    *   Skills (Sub-section)
    *   Education (Sub-section)
*   **Resume**
*   **Contact**

## 2. Navigation Structure (Global)
*   **Primary Navigation (Header):**
    *   Projects
    *   Experience
    *   About
    *   Contact
    *   [Resume - CTA Button]

## 3. Content Hierarchy
*   **Level 1 (Top Level):** Home, Projects, Experience, About, Resume, Contact
*   **Level 2 (Detail Level):** Individual Project Pages (linked from Projects)

## 4. Internal Linking Strategy
*   **Contextual Links:**
    *   "Skills" on About page should link to "Projects" demonstrating those skills.
    *   "Experience" page should link to specific "Projects" achieved in those roles.
    *   "Home" page should feature key "Projects" and link to the full "Projects" page.
*   **Call to Action (CTA):** All pages should contain a subtle footer link to "Contact" and "Resume".

## 5. Recruiter User Journey
1.  **Home** (Hero identifies purpose) ->
2.  **Projects** (Scanning for relevant experience) ->
3.  **Resume** (Quick access to PDF) ->
4.  **Contact** (Action to reach out)

## 6. Client User Journey
1.  **Home** (Introduction/Approach) ->
2.  **About** (Establish trust/values) ->
3.  **Projects** (Deep dive into case studies) ->
4.  **Contact** (Inquiry)

## 7. Mobile Navigation Strategy
*   **Persistent Header:** Logo + Hamburger Menu.
*   **Hamburger Menu:** Full-screen overlay or slide-out drawer containing all global navigation links + Resume CTA.

## 8. Footer Structure
*   **Copyright:** © [Year] [Name]
*   **Quick Links:** Home, Projects, About, Contact
*   **External Links:** GitHub, LinkedIn, Email
*   **Secondary:** Privacy Statement/Terms (if applicable)
